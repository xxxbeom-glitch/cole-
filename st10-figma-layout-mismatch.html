ST-10 Figma 레이아웃 불일치 - Claude 문의용

반원형 게이지와 애니메이션은 정상 동작합니다. 문제는 텍스트 크기/스타일, 요소 간 간격, 버튼 위치가 Figma 디자인과 맞지 않다는 것입니다.

2. 불일치하는 영역 (게이지 제외)

현재 구현과 Figma가 다른 부분

  • 텍스트: 헤더("장원영 님의" / "스마트폰 중독 테스트" / "결과입니다") 폰트 크기, 굵기, 줄 간격
  • 요약 메시지: "스마트폰 사용 습관이 건강해요!" 크기·스타일
  • 습관 영역: 항목별 간격, 라벨/상태 텍스트 크기
  • 디스클레임: 폰트 크기, 색상, 간격
  • 버튼 영역: "확인" 버튼 위치·크기·라운드, "다시하기" 텍스트 위치·간격
  • 전체 간격: 상단~헤더, 헤더~게이지, 게이지~요약, 요약~습관, 습관~디스클레임, 디스클레임~버튼, 버튼~다시하기, 하단 패딩

5. 요청 내용

Figma Dev Mode 또는 디자인에서 317-3002 노드의 다음 값을 확인하고, Android Compose(dp/sp)로 변환한 값을 알려주세요:

  • 각 텍스트 요소의 font-size(px), font-weight, line-height
  • 요소 간 gap/spacing (px → dp)
  • 버튼 height, border-radius, padding
  • "확인" 버튼과 "다시하기" 사이 간격

또는 위 Figma 링크를 참고해 수정된 Kotlin 코드 (SelfTestResultScreenST10.kt의 ST10Spacing, Text style, 버튼 영역)를 제공해 주세요.
