구독 안내 화면(SubscriptionGuideScreen) 구현 문제점 정리

목적: Android Compose 앱의 구독 유도 전체화면. Figma MA-01 디자인 기준으로 구현했으나 여러 문제가 발생함. 원인 분석 및 수정 방향을 Gemini에게 문의하기 위한 정리.

1. 닫기(X) 버튼 아이콘 문제

  • 문제: 유저가 제공한 PNG 파일(흰 원 + 검정 X)이 적용되지 않음. 현재는 코드로 만든 Vector Drawable만 사용 중.
  • 제공된 파일: close-c1d28989-2687-4deb-aa54-a3f0b3519604.png (또는 유사 경로의 close PNG)
  • 현재 구현: ic_subscription_close.xml - Vector drawable (흰 원 + stroke로 그린 X)
  • 의문: PNG를 drawable로 넣어야 하는데, 어떤 drawable 폴더/이름 규칙으로 넣고, Compose에서 어떻게 참조해야 하는지? 또는 Vector가 맞는 접근인지?

2. 체크뱃지(체크 아이콘) 문제

  • 문제: 기능 목록 앞에 쓰는 보라색 체크뱃지 PNG가 적용되지 않음.
  • 제공된 파일: Group_515-d1c4d06d-62c7b094-4f22-4f4f-a227-474453f01fd1.png (purple gear-like shape with black checkmark)
  • 현재 구현: ic_subscription_check_badge.xml - 단순 보라색 원+체크 Vector
  • 의문: 유저 제공 PNG를 drawable로 적용하는 정확한 방법?

3. 구독 카드 캐러셀 문제

  • 문제: 화면에 카드 1장만 보이고, 스와이프해도 캐러셀처럼 동작하지 않음. 인디케이터(점 2개)는 보이나 실제로 2개 카드가 보이거나 스와이프되는 느낌이 없음.
  • 기대: 연간/월간 두 카드가 캐러셀 형태로 나열되고, 좌우 스와이프로 전환 가능.
  • 현재 구현: HorizontalPager 사용, pageCount=2, pageSpacing=10.dp. 각 페이지에 SubscriptionPlanCard 1개씩.
  • 의문: HorizontalPager가 스크롤 내부(verticalScroll Column)에 있어서 제스처 충돌? 또는 높이/너비 제약으로 한 페이지만 보이는 구조? 캐러셀 패턴(한 카드가 화면 대부분 차지, 옆 카드가 살짝 보이는 식)을 위해선 LazyRow 또는 다른 레이아웃이 나을지?

4. 폰트 적용 문제

  • 문제: Figma와 동일한 폰트/굵기/사이즈가 적용되지 않는 것 같다는 피드백.
  • 현재 구현: SubscriptionTypography 객체로 SuitVariable + FontWeight 하드코딩 (ExtraBold 26sp, SemiBold 16sp, Bold 18sp 등).
  • 의문: SuitVariable Variable Font에서 FontWeight만 지정했을 때 wght axis가 제대로 반영되는지? 프로젝트 AppTypography에서는 FontVariation.weight()를 쓰는데, SubscriptionTypography에서는 fontWeight만 사용함.

5. CTA 버튼 문제

  • 문제: "내가 준대로" 적용이 안 됐다는 피드백. 구체적인 차이는 불명.
  • 현재 구현: 하단 전체 너비, 높이 78dp, Primary300 배경, "지금 시작하면 7일은 무료" 텍스트(SubscriptionTypography.CtaButton).
  • 의문: Figma와 다른 점(높이, 라운드, 텍스트 스타일, 패딩 등)이 있는지 확인 필요.

환경

  • Android, Jetpack Compose, Kotlin
  • Figma 디자인: MA-01 구독 유도 화면
  • 파일: SubscriptionGuideScreen.kt, ic_subscription_close.xml, ic_subscription_check_badge.xml

함께 첨부해야 할 파일

1. app/src/main/java/com/cole/app/SubscriptionGuideScreen.kt
2. app/src/main/res/drawable/ic_subscription_close.xml
3. app/src/main/res/drawable/ic_subscription_check_badge.xml
4. app/src/main/java/com/cole/app/AppTypography.kt (SuitVariable, FontVariation 참고용)
5. 유저 제공 PNG: close 버튼, 체크뱃지 (채팅 시 첨부했던 이미지 - Cursor assets에 저장됐을 수 있음. 재첨부 또는 경로 공유)
6. (선택) Figma 스크린샷 또는 디자인 링크

Gemini에게 요청할 핵심 질문

  • 1) HorizontalPager가 verticalScroll 안에서 제대로 동작하지 않는 이유와, 캐러셀 형태로 두 카드를 스와이프해서 보이게 하는 올바른 구현 방법
  • 2) 유저 제공 PNG를 drawable로 추가하고 Compose에서 사용하는 정확한 절차
  • 3) SuitVariable 같은 Variable Font에서 FontWeight 적용 시 FontVariation 설정이 필요한지, SubscriptionTypography에서 올바른 적용 방법
