# 통계 화면 데이터 규칙

> `StatisticsData.kt`, `StatisticsScreen.kt` 기준 적용

## x축 구성

- **3시간 간격**: 3, 6, 9, 12, 15, 18, 21, 24
- **24**: 자정 구간 (00:00~02:59) 의미

## 구간 정의

| 라벨 | 시간 범위 |
|------|-----------|
| 24 | 00:00 ~ 02:59 |
| 3  | 03:00 ~ 05:59 |
| 6  | 06:00 ~ 08:59 |
| 9  | 09:00 ~ 11:59 |
| 12 | 12:00 ~ 14:59 |
| 15 | 15:00 ~ 17:59 |
| 18 | 18:00 ~ 20:59 |
| 21 | 21:00 ~ 23:59 |

## 탭별 데이터 기간

| 탭 | 기간 |
|----|------|
| 오늘 | 오늘 00:00 ~ 현재 시각 |
| 주간 | 이번 주 월요일 00:00 ~ 현재 시각 |
| 월간 | 이번 달 1일 00:00 ~ 현재 시각 |
| 연간 | 올해 1월 1일 00:00 ~ 현재 시각 |

## 그래프 규칙

- **현재 시각이 속한 구간까지만 바 표시**
- 아직 오지 않은 미래 구간은 바 없음 (빈 칸)
- 미래 구간은 터치 비활성 (`pointerInteropFilter`로 이벤트 소비)

### 현재 시각별 표시 예시 (오늘 탭)

| 현재 시각 | 표시 구간 | 빈 칸 |
|-----------|-----------|-------|
| 00:09 | 24 | 3~21 |
| 08:30 | 24, 3, 6 | 9~21 |
| 14:00 | 24, 3, 6, 9, 12 | 15~21 |
| 23:59 | 모든 구간 | 없음 |

## 최다 앱 리스트 규칙

- **그래프와 동일한 기간** 데이터 기준
- 탭 전환 시 **그래프 + 리스트 동시 갱신**
- 해당 기간 내 사용시간 **내림차순** 정렬 (최대 5개)
- 시스템 앱 제외

## 공통 규칙

- 그래프와 최다 앱 리스트는 **항상 같은 기간** 데이터를 사용
- 탭 변경 시 두 영역 모두 동시 업데이트
- `LaunchedEffect(tabEnum)` 기반으로 동시 로드
